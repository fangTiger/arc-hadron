## Context

HADRON Web 使用 Next.js 16、React 19 与 Tailwind 4，页面视觉以深空黑、霓虹青、小号 mono 数据和紧凑交易密度为核心。现有页面分别使用 `max-w-7xl`、`max-w-[1440px]` 与局部固定宽度；顶栏在 `sm` 以下隐藏主导航，却仍同时展示品牌、Assistant、网络和钱包；数据表的最小宽度未被所有父级约束。真实浏览器审计显示 390px 视口被全局顶栏撑为 447px，issuer 页进一步被资产表撑为 998px，首页固定 ticker 覆盖内容，长文档和 loading skeleton 在窄屏上也缺少合适尺度。

本次工作只调整表现层与布局契约。交易 hooks、链上数据、过滤参数、排序行为、API 内容和钱包语义保持不变。

## Goals / Non-Goals

**Goals:**

- 保留现有暗色交易终端身份，统一跨页面的 gutter、最大宽度、垂直节奏和表面层级。
- 在 390px、768px、1280px 和 1440px 关键视口下消除页面级横向溢出。
- 让顶栏在手机上完整可用，让数据表只在自身容器内滚动，让 LIVE ticker 不遮挡主内容。
- 提升 developer API、空状态和加载态的阅读舒适度，同时保持市场与交易页的紧凑密度。
- 通过源码契约测试、现有组件测试、构建和真实浏览器截图形成回归证据。

**Non-Goals:**

- 不做品牌重塑、浅色主题或营销型 hero。
- 不改变页面信息架构中的“市场数据优先、交易 rail 优先”顺序。
- 不新增组件库、图标库或动画依赖。
- 不修改合约、API、钱包、链上读取和业务状态机。

## Decisions

### 1. 采用“统一壳层 + 页面密度变体”，不做逐页补丁

在 `globals.css` 中定义页面最大宽度、响应式 gutter、顶栏尺寸和 ticker 安全区等 token，并让各路由共享同一 shell 对齐线；数据密集页面保留 1440px 工作区，文档/portfolio 可在 shell 内使用更窄阅读列。

备选方案一是只修当前截图中暴露的 class，改动少但会继续产生新的宽度漂移；备选方案二是建立完整 Design System/组件库，长期收益较高但超出本次修复范围。统一壳层是风险和一致性的平衡点。

### 2. 移动顶栏采用两层紧凑布局

小于 `sm` 时，第一层保留品牌与核心操作，网络状态缩为不会抢占宽度的短标签；第二层提供可访问的 MARKET / PORTFOLIO / API 导航。`sm` 以上恢复单行桌面结构。该方案无需新菜单状态，也不会把导航完全隐藏。

备选的汉堡菜单会增加焦点管理、点击外部关闭和动画状态；单纯缩小字体仍无法容纳现有五组内容，因此不采用。

### 3. 表格溢出只允许发生在局部滚动框

所有含宽表格的 grid item 必须具备 `min-width: 0`；滚动框使用 `max-width: 100%` 与 `overflow-x: auto`，内部表格保留可读的最小宽度。窄屏增加边缘渐隐或简短滚动提示，让用户知道仍有列可查看。页面根节点增加保守的横向裁剪作为最后防线，但不能用它掩盖错误的父级约束。

把表格强行压成 390px 会让 8 列数据不可读；把每行改为卡片会改变交易所信息密度与排序比较方式，因此不采用。

### 4. 固定 ticker 只用于有足够高度的宽屏

桌面保留固定底部 tape，并通过 shell 安全区为其预留空间；窄屏 ticker 改为主内容末尾的普通流式区块，避免永久覆盖表格和交互。无事件时仍保留既有 LIVE 状态语义，不制造虚假活动。

### 5. 通过阅读密度而非全局放大改善美观

市场数字与标签继续使用小号 mono；文档正文、说明文字、空状态和卡片标题使用更舒适的字号与行高。边框、背景、强调色继续复用现有 token，新增的视觉深度只通过轻微表面色差、分区留白和 hover/focus 状态实现，避免霓虹光晕泛滥。

### 6. 测试布局契约，不锁死像素截图

先用 Vitest 对关键源码/静态标记建立失败测试：移动导航存在、shell 与局部滚动标记存在、ticker 响应式策略存在、loading 采用响应式高度。实现后运行现有测试与构建，再用真实 Chrome 在关键路由和视口检查 `scrollWidth === innerWidth`，并保存截图作为人工视觉证据。

## Risks / Trade-offs

- [两层移动顶栏增加首屏高度] → 控制第二层高度与 padding，并通过更清晰导航换取可达性。
- [局部横向滚动隐藏部分列] → 保持资产名称/价格为首列，提供边缘提示并支持触控滚动。
- [全局 shell 改动影响多页] → 先增加契约测试，分批替换页面壳层，每批运行相关测试。
- [远程链读取导致截图停留在 skeleton] → 同时验证 loading 布局和静态 DOM 契约；有数据时再检查完整内容，不把外部 RPC 成功作为布局验证前提。
- [未提交的用户文件较多] → 仅修改 proposal/tasks 中列出的 Web 与 OpenSpec 文件，不触碰无关本地状态。

## Migration Plan

1. 先增加响应式布局契约测试并确认按预期失败。
2. 实现全局 shell 与顶栏，再修市场/issuer 的局部滚动和 ticker。
3. 调整 developer API、portfolio 与资产 loading 的阅读和占位尺度。
4. 运行相关测试、全量测试、lint、build 与关键视口浏览器回归。
5. 若出现回归，可按模块回退 class 与 CSS token；本变更不涉及数据迁移。

## Open Questions
